<h1><a href="/entry/bsides-vancouver-2018-workshop,231/">BSides Vancouver: 2018 (Workshop)</a></h1>

<div class="row-fluid hDiv"></div>

<a href="/entry/bsides-vancouver-2018-workshop,231/">
<figure>
<img class="img-polaroid" src="/media/img/author/BSides-Vancouver-2018-Workshop-01-crop.png" alt="BSides Vancouver 2018 (Workshop) screenshot" height="288" width="150" title="Screenshot of BSides Vancouver 2018 (Workshop)" />
</figure>
</a>
</div>
<div class="right"> 
<p>Boot2root challenges aim to create a safe environment where you can perform real-world penetration testing on an (intentionally) vulnerable target.</p>
<p>This workshop will provide you with a custom-made VM where the goal is to obtain root level access on it. </p>
<p>This is a great chance for people who want to get into pentesting but don’t know where to start. *</p>
<p>If this sounds intimidating, don’t worry! During the workshop, we’ll be discussing various methodologies, common pitfalls and useful tools at every step of our pentest.</p>
<p>Requirements:</p>
<ul>
<li>Laptop capable of running two VMs and has a USB port.</li>
<li>At least 20GB of free space.</li>
<li>VirtualBox pre-installed.</li>
<li>Kali VM</li>
<li>Some familiarity with CLI.</li>
</ul>

<h1><a href="/entry/trollcave-12,230/">Trollcave: 1.2</a></h1>
</div>
<div class="span3 auth-date">
<h5 class="pull-right">
<a href="/author/david-yates,580/">
<span class="author"> David Yates</span>
</a> <small>21 Mar 2018</small>
</h5>
</div>
</div>
<div class="row-fluid hDiv"></div>
<div class="row-fluid entry-content">
<div class="span2 left"> 
<a href="/entry/trollcave-12,230/">
<figure>
<img class="img-polaroid" src="/media/img/entry/Trollcave-01-crop.png" alt="Trollcave 1.2 screenshot" height="288" width="150" title="Screenshot of Trollcave 1.2" />
</figure>
</a>
</div>
<div class="right"> 
<p><strong>Trollcave</strong> is a vulnerable VM, in the tradition of <a href="https://www.vulnhub.com/">Vulnhub</a> and <a href="https://en.wikipedia.org/wiki/Wargame_(hacking)">infosec wargames</a> in general. You start with a virtual machine which you know nothing about – no usernames, no passwords, just what you can see on the network. In this instance, you'll see a simple community blogging website with a bunch of users. From this initial point, you enumerate the machine's running services and general characteristics and devise ways to gain complete control over it by finding and exploiting vulnerabilities and misconfigurations.</p>
<p>Your first goal is to abuse the services on the machine to gain unauthorised shell access. Your ultimate goal is to read a text file in the <code>root</code> user's home directory <code>root/flag.txt</code>).</p>
<p>This VM is designed to be holistic and fairly down to earth. I wanted to simulate a real attack on a real website rather than just presenting a puzzle box of disparate elements, and I wanted to avoid the more esoteric vulnerable VMisms, like when you have to do signal processing on an MP3 you found to discover a port-knocking sequence. Of course there are always tradeoffs between what's realistic and what's optimally fun/challenging, but I've tried to keep the challenges grounded.</p>
<p>Because this is a VM that you're downloading, importing and booting, one way to achieve this goal would be to mount the VM's hard disk. I haven't encrypted the disk or done anything to prevent this, so if you want to take that route, go ahead. I'm also not offering a prize or anything for completing this VM, so know that it will be entirely pointless.</p>
<p>Because this is a VM running a real operating system with real services, there may be ways to get to <code>root</code> that I did not intend. Ideally, this should be part of the fun, but if they make the box entirely trivial I'd like to know about and fix them – within reason. As of this release, I've installed all the updates available for Ubuntu Server 16.04 LTS, but I cannot and will not attempt to patch this VM against every new Linux kernel exploit that comes out in the future. So there's a hint – you don't have to use a kernel exploit to root this box.</p>
<p>What you will need is a good HTTP intercepting proxy – I recommend <a href="https://portswigger.net/burp">Burpsuite</a> – and a couple of network tools like <code>nmap</code> and <code>nc</code>. You'll also need some virtualisation software – <a href="https://www.virtualbox.org/wiki/Downloads">VirtualBox</a> will be easiest for most people, but KVM and VMWare should also be able to import the <code>.ova</code> file after a bit of fiddling. Once you've imported the VM, put it on the same network as your attacking system (<a href="https://digi.ninja/blog/untrusted_vms.php">preferably don't give it internet access</a>) and start hacking!</p>
<p>You can grab the <code>.ova</code> file here (<a href="https://drive.google.com/open?id=1DhbVvT-r3yKWXXjcH70oj5yEZ88TDsky">929MB</a>) (updated 2018-03-19). Let me know <a href="https://davidyat.es/cdn-cgi/l/email-protection#3e5a7e48575a475f4a105b4d014d4b5c545b5d4a036a4c5152525d5f485b">what you think</a>.</p>
</div>
